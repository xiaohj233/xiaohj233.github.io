为了一碟醋，我包了一顿饺子

晓嘿君2025-07-192025-09-05

久违的更新：为了一碟醋，包了一顿饺子

暑假已经过去四分之一了，才发现已经这么久没有更新博客了。久违的来更新一下，这次就不用 AI 来糊弄大家了，纯手工打造的。

一个叫 Flashback 的模组

最近几天一直在忙一件事 —— 给一个叫 flashback 的我的世界模组做汉化。这模组是搞回放录制的，但跟我用过的其他回放模组比，他有个其他模组做不到的地方：能直接编辑回放里的内容，自由度高到离谱。

巧的是，我最近刚好有做视频的想法，这模组能完全能做到我新视频想要达到的效果。

不过这模组刚出来不到一年，好多功能还没完善，连中文汉化都没有。但是我认为它潜力确实大，而且我之前用过这作者做的另一个模组，质量也是相当可以。所以我毅然决然决定给他添加 i18n 汉化。


这里插个小插曲：因为之前有人倒卖作者的模组，作者在开源协议里特意保留了所有权。所以一开始我也不确定，不知道作者会不会接受我的代码。不过当时想的是就算不接受，那我自己用就行——毕竟主要还是为了方便我视频。当然了，要是能通过，也算是给开源社区尽了一份力。

具体是怎么做的？

说起来有点抽象。我基本没写过 Java 代码，就只对 Java 有个粗略的了解。

为了汉化，我可谓是神农尝百草，把各家带 AI 功能的 IDE 都试了个遍。论体验，Claude Code 确实最好，但架不住它贵啊。最后还是靠 Cursor 当主力军，谁让它能白嫖呢。

这几天的流程大致是这样：

  1. 先给 AI 提要求，然后鞭策它写代码；
  2. 测完发现不对，再鞭策它改；
  3. 不满意就退回去重新鞭策，再测；
  4. 直到满意了再写下一个，纯纯把AI当工具人（）。

这几天下来，自己的编码能力没提升多少，写提示词的能力倒是越发熟练，感觉以后自己适合去当甲方（滑稽）。

就这么折腾了三天三夜，总算靠 “鞭策” AI 把汉化搞定了，并且尽可能的修复了我遇到的所有问题，做到了最小化更改。

作者怎么说？

搞定汉化后，我便让 AI 帮我写一段给原作者的英文信，然后抱着忐忑的心情去私信找作者交谈。主要是怕被拒——毕竟作者之前表过态；还有就是这是我第一次和外国人交谈，现在想起来感觉也算非常不可思议，居然是因为这事。

接着我做好了完全的准备，准备好了文字，准备好了文件，准备好了被接受或被拒绝的心理准备，发送了文章。结果才过了不到十秒，作者就回了句：

“Sure, no problem.”

那一刻，忐忑的心情瞬间烟消云散，转而是一种无形的喜悦。

（不过这里就不升华了，要升华也是留到写作文的时候用（滑稽）。）

后来作者还发了条消息，说他本来就有支持其他语言的计划，只是时候未到。我把文件发过去，给他表达了感谢后，这事就算告一段落。如果后续有什么消息，我再续写该文章。

后续更新（2025年8月22日）

好消息！就在不久前，作者发布了新版本0.37.0，正式采纳并整合了我提交的汉化工作！

现在，《Flashback》模组的最新版本已经自带了完整且正常显示的中文支持。所有中文玩家都可以直接去官方页面（CurseForge/Modrinth）下载，无需再等待或使用任何非官方版本。

你的每一份贡献，都可能成为推动项目变得更好的重要一环。

接下来呢？

这时候可能有人会问，做完这么大个工程，接下来干什么呢？

其实我早有安排。暑假刚开始的那段时间，我还自己写了个小模组，不过目前只实现了部分功能，还需要接下来几天继续完善。当然具体内容目前保密，到时候的新视频里你可能就可以看到他的身影。

不过说起来也是神奇，不管是做这个汉化，还是弄我那个小模组，说到底都是为了一个事情——制作新视频。可谓是为了一碟醋包了一顿饺子，虽然我也并不是第一次干这种事情了。

至于视频什么时候更新，我之前在B站动态里说了，两周内（截止文章发布还剩9天）。不得不说我B站动态发的比我博客发的勤快。

写的也差不多了，就这样吧。

如果你们也有“为了一碟醋包了一顿饺子”的事情，也可以分享在评论区（狗头）。
